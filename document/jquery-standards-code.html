    加载jQuery

    1. 我们经常通过CDN技术方式加载jQuery。CDN 的优势 
      <script type="text/javascript" src="//ajax.googleapis.com/ajax/libs/jquery/1.11.0/jquery.min.js"></script>
<script>window.jQuery || document.write('<script src="js/jquery-1.11.0.min.js" type="text/javascript"><\/script>')</script>
      点击这里查看 推荐流行的CDN服务.

      比较好的jQuery CDNs

      Google: https://developers.google.com/speed/libraries/devguide
      Microsoft: http://www.asp.net/ajaxlibrary/cdn.ashx
      jQuery: http://jquery.com/download/
      cdnjs: http://cdnjs.com
      OSSCdn: http://osscdn.com
    2. 同时我们可以像上面代码使用一个本地同版本库备份文件。 更多信息
    3. 上面文件引入路径使用协议相对/协议独立URL（注意URL中去除http: 或 https:）
    4. 如果可能尽量将jQuery引入script标签放在页面的底部。更多信息 基于 HTML5 Boilerplate模板.
    5. 版本的使用情况？
      • 如果要支持IE6/7/8时请不要用jQuery 2.X版本。
      • 当我们开发webapp时，并不需要考虑太多兼容性问题就尽量使用更高版本的jQuery。
      • 使用jQuery的CDN服务时，要注意区分jQuery的引入版本。 (比如: 1.11.0 跟 1.11 一样都是 jQuery1.x最高版本).
      • 千万不要使用多个jQuery版本共存。
    6. 如果你同时使用了Prototype, MooTools, Zepto等库时，最好使用jQuery替代$。在jquery函数内部使用时记得调用$.noConflict()。
    7. 要检测现代浏览器更高版本功能，请使用 Modernizr库。

    jQuery变量

    1. 所有的变量应该定义一个$开头的变量名储存jQuery对象。
    2. 用变量存储经常使用的jQuery对象。 
      var $myDiv = $("#myDiv");
$myDiv.click(function(){...});
    3. 变量名最好使用驼峰方式。

    选择器

    1. 无论什么地方使用ID选择器时我们可以直接使用document.getElementById()获取对象，这样执行效率更高。
    2. 无论什么时候使用类选择器时不要加上元素的类型（标签） 表达式的执行效率 
      var $products = $("div.products"); // 慢
var $products = $(".products"); // 快
    3. 在ID-子潜逃类型选择器时尽量用find()函数。.find()函数执行速度更快因为不用经过Sizzle引擎调用。更多Sizzle性能信息 
      // 糟糕的，需经过Sizzle库选择器混合处理
var $productIds = $("#products div.id");

// 不错的，#productst通过原生document.getElementById()获取上下文范文后再去使用Sizzle库获取类选择器匹配
var $productIds = $("#products").find("div.id");
    4. 遵从右边选择器更详细原则，左边选择器尽量简单。 选择器优化 
      // 没有优化的
$("div.data .gonzalez");

// 优化了的
$(".data td.gonzalez");
    5. 避免复杂的选择器表达式。选择器优化, 表达式性能 
      $(".data table.attendees td.gonzalez");
 
// 更好的方式，减少中间匹配过程
$(".data td.gonzalez");
    6. 给定选择器的上下文执行环境（作用作用域） 
      // 比较慢的，引文要变量DOM所有匹配.class的元素
$('.class');

// 更快的，只需在ID class-container元素子元素中查找
$('.class', '#class-container');
    7. 避免通用选择符。 选择器优化 
      $('div.container > *'); // 糟糕的
$('div.container').children(); // 更好的
    8. 避免不明的伪类选择器，当少了标签是通配符一直不明确选择器优化 
      $('div.someclass :radio'); // 糟糕的
$('div.someclass input:radio'); // 更好的
    9. 不要使用多个ID选择器或组合的ID选择器。因为ID选择是通过原生的document.getElementById()方法获取，性能更快更好 
      $('#outer #inner'); // 糟糕的
$('div#inner'); // 糟糕的
$('.outer-container #inner'); // 糟糕的
$('#inner'); // 更好的, 只是调用了document.getElementById()方法

    DOM操作

    1. 在操作DOM元素之前将其从页面分离。 元素分离 
      var $myList = $("#list-container > ul").detach();
// 一连串对$myList进行操作
$myList.appendTo("#list-container");

      我的理解是该函数将DOM元素拷贝一份出来，然后在该对象上进行操作

    2. 在做字符串的连接是使用数组的array.join() 的方法替换 .append()函数操作。append在循环外部的优势
      性能比较: http://jsperf.com/jquery-append-vs-string-concat 
      // 糟糕的
var $myList = $("#list");
for(var i = 0; i < 10000; i++){
    $myList.append("<li>"+i+"</li>");
}
 
// 不错的
var $myList = $("#list");
var list = "";
for(var i = 0; i < 10000; i++){
    list += "<li>"+i+"</li>";
}
$myList.html(list);
 
// 最快最好的
var array = []; 
for(var i = 0; i < 10000; i++){
    array[i] = "<li>"+i+"</li>"; 
}
$myList.html(array.join(''));
    3. 不要跟不存在的DOM元素使用动画函数更多信息 
      // 糟糕的: 执行了三次，还元素不存在报错
$("#nosuchthing").slideUp();
 
// 不错的
var $mySelection = $("#nosuchthing");
if ($mySelection.length) {
    $mySelection.slideUp();
}

    事件处理

    1. 每个页面应该只有一个Dom ready回调操作，利于我们测试和bug跟踪。
    2. 不要使用为事件绑定匿名函数，因为匿名函数不利我们的调试和不能够重用代码。More Info 
      $("#myLink").on("click", function(){...}); // 糟糕的
 
// 不错的
function myLinkClickHandler(){...}
$("#myLink").on("click", myLinkClickHandler);
    3. 同样的，Dom Ready不建议使用匿名函数绑定 
      $(function(){ ... }); // 不能为dom ready写
 一个测试案例
// GOOD
$(initPage); // 或者 $(document).ready(initPage);
function initPage(){
    // 页面初始化，可以还调用其他的函数.
}
    4. 页面DOM ready就绪操作应该放到外部js文件。 
      <script src="my-document-ready.js"></script>
<script>
	// 配置变量全局
	$(document).ready(initPage); // or $(initPage);
</script>
    5. 不要在html标签内使用事件标签，通过jQuery绑定我们更多容易绑定或解除事件 
      <a id="myLink" href="#" onclick="myEventHandler();">my link</a> <!-- BAD -->
      $("#myLink").on("click", myEventHandler); // GOOD
    6. 尽量可能使用为事件绑定使用事件命名空间。这样更简单移除DOM元素特定的事件操作。 
      $("#myLink").on("click.mySpecialClick", myEventHandler); // GOOD

// 只移除click.mySpecialClick事件，并不影响该元素其他的click事件
$("#myLink").unbind("click.mySpecialClick");

    Ajax

    1. 避免使用.getJson() 或者 .get()，虽然是$.ajax()函数简写方式。
    2. 不要在https的页面使用http请求。
    3. 不要在请求URL中添加参数，尽量使用data配置参数。 
      // 可读性差
$.ajax({
    url: "something.php?param1=test1&param2=test2",
    ....
});
 
// 更好的可读性
$.ajax({
    url: "something.php",
    data: { param1: test1, param2: test2 }
});
    4. 尽量指定dataType的类型，这样更快执行回调的数据。
    5. 使用委托方式绑定加载ajax内容，更利于子元素的处理过程。 More Info 
      $("#parent-container").on("click", "a", delegatedClickHandlerForAjax);
    6. 使用Promise延时接口: 更多例子 
      $.ajax({ ... }).then(successHandler, failureHandler);
 
// OR
var jqxhr = $.ajax({ ... });
jqxhr.done(successHandler);
jqxhr.fail(failureHandler);
    7. Ajax使用模板: 更多信息 
      var jqxhr = $.ajax({
    url: url,
    type: "GET", // 默认是GET
    cache: true, // 默认是TRUE，但当dataType是'script' 和 'jsonp'是false
    data: {}, // 请求参数
    dataType: "json", // 返回处理数据类型
    jsonp: "callback", // 指定jsonp的回调函数名
    statusCode: { // ajax状态处理
        404: handler404,
        500: handler500
    }
});
jqxhr.done(successHandler);
jqxhr.fail(failureHandler);

    动画特效

    1. 使用动画的方式尽量统一一致。
    2. 不要为了用户体验重写一遍动画效果。
      • 为元素做toggle开关使用show/hide, toggle 和 slideUp/slideDown 函数。
      • 使用动画时间过渡为 "slow", "fast" 或 400 。

    插件

    1. 通常选择一个具有好的技术，文档，测试和社区支持的插件。
    2. 要考虑插件与你使用的jQuery版本兼容性问题。
    3. 按照jQuery公用模板方式编写你的插件。插件模板

    链式定义

    1. 使用链式方式能缓存选择器对象 
      $("#myDiv").addClass("error").show();
    2. 无论什么时候超过3个链式时换行并对齐，这样更好的代码可读性。 
      $("#myLink")
    .addClass("bold")
    .on("click", myClickHandler)
    .on("mouseover", myMouseOverHandler)
    .show();
    3. 更长的链式能只是缓存了一个对象变量。

    其他

    1. 参数尽量用对象字面量 
      $myLink.attr("href", "#").attr("title", "my link").attr("rel", "external"); // 糟糕的，调用了3次attr()
// 不错的，只调用了一次attr()
$myLink.attr({
    href: "#",
    title: "my link",
    rel: "external"
});
    2. 不建议用jQuery的css函数定义样式 
      $("#mydiv").css({'color':red, 'font-weight':'bold'}); // 糟糕的
      .error { color: red; font-weight: bold; } /* 不错的 */
      $("#mydiv").addClass("error"); // 不错的，通过增加和移除样式名来改变dom样式
    3. 不要使用废弃的jQuery方法。 查看 废弃方法。
    4. 必要时考虑使用原生javascript语法，性能更快一些。 http://jsperf.com/document-getelementbyid-vs-jquery/3 
      $("#myId"); // 始终比原生的稍慢...
document.getElementById("myId");
